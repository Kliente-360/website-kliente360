# Auditoria de design system — divergências entre páginas

> Onda 1 do refator de consistência. Inventário de duplicações antes de decidir o esforço de unificação.
> Estado: PLAN.md §4.4 ainda lista pillar pages + glossário + comercial como "entregue", mas elas trazem dívida técnica de design system.
> Data: 2026-05-22 (quinta passada).

## Resumo executivo

- **`assets/css/main.css`** tem 145 regras de classe.
- **Eyebrow** tem **6 implementações distintas** com tratamentos diferentes (cor, peso, espaçamento, antes/depois).
- **`section-head`** está duplicado em `.cm-section`.
- **Padding-block** de seções segue **4 escalas diferentes** sem regra clara.
- **Tamanhos de H2** divergem em pelo menos 6 contextos sem alinhar com o token `--fs-h2`.
- **Cards** existem em 9 variantes (`.pillar-card`, `.pp-offer-card`, `.cm-mode`, `.gloss-term`, `.case-card`, `.post-card`, `.pp-stat`, `.cm-expect-item`, `.pp-trilha-step`). Pelo menos 4 poderiam ser variações de um mesmo componente.

## Inventário de divergências

### 1. Eyebrow — 6 implementações para o mesmo papel

| # | Seletor | Linha | Cor | Peso | Tamanho | Notas |
|---|---|---|---|---|---|---|
| 1 | `.eyebrow` (base) | 20 | `--color-accent` (verde UI) | semibold | 12px | Versão original; usada na home. |
| 2 | `.section.dark .eyebrow` | 33 | `--green-300` | (herdado) | (herdado) | Sobreposição pra fundo escuro. |
| 3 | `.pillar-deep .eyebrow` | 376 | cor do pilar | semibold | (herdado) | **Vira pílula** com fundo tintado + `::before` dot. Não é mais eyebrow textual. |
| 4 | `.pp-hero .heading .eyebrow` | 938 | `rgba(255,255,255,0.7)` | semibold | 12px | Pillar page hero (fundo colorido). |
| 5 | `.pp-manifesto/offers/trilha/posts .eyebrow` | 1009/1045/1106/1170 | cor do pilar | semibold | 12px | **4 cópias idênticas** dentro de pillar pages. |
| 6 | `.pp-cta .eyebrow, .cm-cta .eyebrow` | 1200 | `--green-300` | semibold | 12px | CTA escura final (unificado agora). |

**Quanto poderia ser uma classe só com modificadores:** 1 base + 3 variantes (`.eyebrow--on-dark`, `.eyebrow--on-pillar`, `.eyebrow--pill`). Hoje são 6 versões hardcoded.

### 2. Section-head — duplicado

| Seletor | Linha | Diferença |
|---|---|---|
| `.section-head` | 299 | Base global. `max-width: 60ch`, `margin-bottom: sp-12 → sp-16`. |
| `.cm-section .section-head` | 1286 | **Idêntico**, mas redeclarado. |
| `.cm-section .section-head h2` | 1287 | H2 com clamp próprio diferente do `.h2` global. |

**`.cm-section .section-head` é redundante.** Cleanup direto: remover, reusar `.section-head` global.

### 3. Padding-block — 4 escalas sem regra

| Seletor | Mobile | Tablet (≥768) | Desktop (≥1024) |
|---|---|---|---|
| `.section` | sp-16 | sp-24 | sp-32 |
| `.cm-section` | sp-16 | sp-24 | *(falta sp-32)* |
| `.pillar-deep` | sp-16 | sp-24 | *(falta sp-32)* |
| `.pp-offers / .pp-trilha / .pp-posts` | sp-16 | sp-24 | *(falta sp-32)* |
| `.pp-hero` | sp-20 sp-16 | sp-32 sp-24 | — |
| `.pp-manifesto` | sp-20 sp-12 | sp-32 sp-16 | — |
| `.pp-cta / .cm-cta` | sp-24 | sp-40 | — |
| `.blog-hero / .post-header` | sp-16 sp-12 | sp-24 sp-16 | — |

**3 padrões coexistem:**
1. *Section "normal"* — sp-16/24/32 (`.section`, e o que deveria ser `.cm-section`/`.pillar-deep`/`.pp-*`).
2. *Hero* — sp-16/24 ou sp-20/32 dependendo da página.
3. *CTA final* — sp-24/40 (maior, certo).

**O risco**: `.cm-section` e `.pillar-deep` perdem o terceiro breakpoint (1024+) — ficam visualmente menores que `.section` em telas largas.

### 4. Tamanhos de H2 — 6 escalas sem token

| Seletor | clamp | Equivalente em px |
|---|---|---|
| `.h2` global | `var(--fs-h2)` = `clamp(2.5rem, 1.8rem + 3vw, 4rem)` | 40 → 64 |
| `.cm-section .section-head h2` | `clamp(1.875rem, 1.5rem + 1vw, 2.5rem)` | 30 → 40 |
| `.pp-hero .heading h1` (H1, mas comparável) | `clamp(2.5rem, 1.8rem + 3.5vw, 4.5rem)` | 40 → 72 |
| `.pp-manifesto h2` | `clamp(1.875rem, 1.5rem + 1.5vw, 2.75rem)` | 30 → 44 |
| `.pp-offers .head h2 / .pp-posts .head h2` | `clamp(1.875rem, 1.5rem + 1vw, 2.5rem)` | 30 → 40 |
| `.pp-cta h2, .cm-cta h2` | `clamp(2rem, 1.5rem + 1.5vw, 3rem)` | 32 → 48 |

**O design system tem `--fs-h1`, `--fs-h2`, `--fs-h3`, `--fs-h4` definidos em `tokens.css`. As novas páginas ignoram esses tokens e definem clamps próprios.**

### 5. Cards — 9 variantes para padrões similares

| Componente | Uso | Padrão estrutural |
|---|---|---|
| `.pillar-card` | Home overview | num + h3 + p + ul + link |
| `.pp-offer-card` | Pillar pages | num + h3 + p + ul |
| `.cm-mode` | Comercial | num + h3 + duration + p + when |
| `.gloss-term` | Glossário | h2 + pill + p + link |
| `.case-card` | Home cases | client + h3 + metric + tag |
| `.post-card` | Blog | meta + pill + h3 + read |
| `.pp-stat` | Pillar stats strip | num + lbl |
| `.cm-expect-item` | Comercial expects | h3 + p (com barra lateral) |
| `.pp-trilha-step` | Trilha applied | num + verb + desc |

**Quase todos seguem o padrão "card com cabeçalho + corpo + footer".** Dá pra extrair 2-3 cards base com modificadores em vez de 9.

### 6. Outros achados menores

- **`.section.dark .eyebrow`** (33) define cor pra modo escuro, mas **`.pp-cta`** não é `.section.dark` — é classe própria. Cria fragmentação.
- **`.cm-section.dark`** (1283) duplica o tratamento de `.section.dark`.
- **Containers narrow**: usados como `.container-narrow` (global) e implícito em outras páginas via `max-width` inline.
- **Hero patterns**: `.hero` (home), `.pp-hero` (pillar), `.blog-hero` (blog), `.post-header` (post page), `.gloss-hero` (glossário), `.cm-hero` (comercial). **6 heros para o mesmo padrão estrutural** (eyebrow + h1 + lead + (opcional cta-row)).

## Recomendação de unificação (proposta de Onda 2)

### Componentes globais a criar/unificar

```css
/* Hero pattern — uma classe + modificadores */
.page-hero { /* padrão de hero */ }
.page-hero--colored { /* fundo colorido (pillar) */ }
.page-hero--dark { /* fundo preto */ }

/* Section pattern */
.section { padding sp-16/24/32 — já existe } 
.section--alt { background: bg-alt }
.section--dark { background: ink-900; color: white; eyebrow → green-300 }
.section--colored { background: var(--pp-color); color: white }

/* Eyebrow — base + modificadores */
.eyebrow { /* já existe */ }
.eyebrow--on-dark { color: green-300 }
.eyebrow--on-color { color: rgba(255,255,255,0.7) }
.eyebrow--pill { /* vira pílula colorida */ }

/* Section head — base, reusar global */
.section-head { /* já existe */ }
/* Remover .cm-section .section-head */

/* Card base */
.card { /* padrão de cartão branco com borda */ }
.card--accent { border-top: 4px solid pillar }
.card--minimal { /* card sem borda, só padding */ }
```

### Migração concreta

1. **Eyebrow**: 6 implementações → 1 base + 3 modificadores. Pillar pages perdem `.pp-manifesto .eyebrow` etc., usam `.eyebrow.eyebrow--on-color` ou `.eyebrow--pill`.
2. **Section-head**: remover `.cm-section .section-head`, usar `.section-head` global. Remover override de H2 no .cm-section.
3. **Padding-block**: aplicar a escala completa `sp-16/24/32` em `.pillar-deep`, `.cm-section`, `.pp-offers`, `.pp-trilha`, `.pp-posts`. Eles ficam visualmente menores em ≥1024px sem isso.
4. **H2 sizes**: usar `--fs-h2` (40→64) ou `--fs-h3` (32→44) em todas as seções, em vez de clamps próprios. Heading da home é `--fs-h2`; das pillar pages deveria ser `--fs-h2` também, não menor.
5. **Cards**: extrair `.card` base; `.pp-offer-card`/`.cm-mode`/`.pillar-card` viram modificadores ou wrappers.
6. **Heroes**: 6 → 2 (page-hero + page-hero--colored). `.hero`, `.blog-hero`, `.post-header`, `.gloss-hero`, `.cm-hero` consolidados.
