01 — Marca

Mark Aperture + wordmark "kliente 360"

A marca é composta por dois elementos canônicos: Mark Aperture — quatro círculos nas posições cardeais (N/E/S/W) com opacidade crescente no sentido horário (0.45 → 0.65 → 0.85 → 1.0) — e o wordmark em lowercase "kliente 360" com espaço. Cor sagrada: #009900. Forma pura sem container — lê "360°" como compasso/lente.

kliente 360

Lockup definitivo — mark à esquerda, wordmark à direita, alinhamento baseline, gap proporcional. Esta é a forma canônica da marca; tudo abaixo são aplicações dela.

Tamanho mínimo legível: 16px. Abaixo disso, perde o gradiente de opacidade e vira borrão.

Lockup canônico em fundos

Wordmark + mark à esquerda. Use sempre o lockup canônico — variações verticais ou compostas só com justificativa específica.

Safe area (clear space)

Mantenha respiração igual à altura do mark (1×) ao redor da marca. Não cole texto, ícone ou edge dentro dessa área.

01b — Marca

Como usar (e como não usar)

Regras canônicas pra preservar a integridade visual da marca em qualquer aplicação — produto, branding, social, slide ou doc interno.

Mark em verde sagrado #009900

Cor canônica do mark. Nunca recolorir — exceto branco sobre fundo escuro/acento.

Não recolorir aleatoriamente

As cores de pilar (azul/âmbar/violeta) só aparecem dentro das suas respectivas seções no produto — nunca no mark da marca-mãe.

Wordmark sempre lowercase

kliente 360

Lowercase com espaço entre "kliente" e "360". Inter Bold / Semibold.

Não inventar variantes do wordmark

KLIENTE360

Sem CAPS, sem hífen, sem juntar "kliente360", sem itálico, sem stretching.

Lockup horizontal por padrão

kliente 360

Mark à esquerda do wordmark. Alinhamento baseline. Gap = sp-3.

Não separar do wordmark sem motivo

kliente 360

Mark e wordmark são um par. Só separar em contextos onde só cabe o mark (favicon, avatar 32×32).

Em fundo escuro: lockup branco

kliente 360

Mark mantém o verde sagrado; wordmark vira branco. Lê melhor que o verde-sobre-preto.

Não usar wordmark verde sobre escuro

kliente 360

Contraste WCAG fraco. Mark verde + texto branco é o combo certo em fundo escuro.

PADRÃO EDITORIAL

Acento verde em parte do heading

A marca abre o hero com um título em duas partes: a primeira em tinta cheia, a segunda em verde --color-accent. É o gestual que sustenta a tagline canônica — "Conhecimento aplicado, como serviço." — e funciona como assinatura visual da Kliente.

✓ FAÇA
  • Em H1 de hero, abertura de seção forte ou capa de slide.
  • Apenas 1 trecho verde por heading.
  • Manter o trecho coeso (palavra ou expressão curta — 2 a 5 palavras).
  • Usar tag semântica <em> (estilo redefinido pra normal + verde).
✗ NÃO FAÇA
  • Múltiplos trechos verdes no mesmo heading.
  • Verde em H4, body ou caption (vira ruído).
  • Verde em heading sobre fundo escuro — usar branco ou suave.
  • Trecho curto demais (1 caractere) ou longo demais (toda a frase).
<!-- HTML -->
<h1>Conhecimento aplicado, <em>como serviço</em>.</h1>

/* CSS — escopo recomendado: só em hero/capa */
.hero h1 em { font-style: normal; color: var(--color-accent); }

08 — Tom de voz

Direto. Técnico. Sem jargão.

Consultoria informada falando com decisor informado. Sem "transformação digital", sem "jornada", sem "experiência fluida".

Usar

  • "CRM, Dados e IA como uma prática única."
  • "Agentes onde já existe dado."
  • "Time enxuto, poucos clientes estratégicos."
  • "Da prova de valor à operação."
  • "Conversa direta com quem entrega."
  • "Sem camadas de account manager."
  • Nomes próprios: Salesforce, Tableau, Agentforce.

Evitar

  • "Transformação digital"
  • "Jornada de sucesso do cliente"
  • "Experiência fluida e omnichannel"
  • "Soluções inovadoras"
  • Auto-elogio explícito ("nossos sócios incríveis").
  • Adjetivos vazios em série.

11 — Reuso em outros apps

Integração Next.js / Vercel

Como importar este design system num projeto Next.js novo. CSS vanilla, sem framework dependency — funciona em qualquer app React/Next.

1 · Copie 3 arquivos

Os tokens, reset e primitivos são auto-contidos. Copia direto pra app/styles/ (ou src/styles/).

app/
└─ styles/
   ├─ tokens.css     # cores, tipografia, espaço, raio, sombra
   ├─ reset.css      # reset mínimo + box-sizing
   └─ main.css       # primitivos (.card, .btn, .grid-cards…)

2 · Import no globals.css

Ordem importa: tokens primeiro (define vars), reset depois, primitivos por último.

/* app/globals.css */
@import "./styles/tokens.css";
@import "./styles/reset.css";
@import "./styles/main.css";

3 · Self-host das fontes

Inter + JetBrains Mono em .woff2. Vão pra public/fonts/. tokens.css já tem o @font-face apontando.

public/
└─ fonts/
   ├─ inter-latin.woff2
   └─ jetbrains-mono-latin.woff2

4 · Preload no layout

Em app/layout.tsx, pre-carregue a Inter pra eliminar FOIT no primeiro paint.

export default function RootLayout({ children }) {
  return (
    <html lang="pt-BR">
      <head>
        <link
          rel="preload"
          href="/fonts/inter-latin.woff2"
          as="font"
          type="font/woff2"
          crossOrigin=""
        />
      </head>
      <body>{children}</body>
    </html>
  );
}

5 · Mark Aperture como componente

Não usa CSS — SVG inline. Cole num components/Mark.tsx e reuse onde precisar.

export function Mark({ size = 32 }) {
  return (
    <svg
      className="mark-aperture"
      viewBox="0 0 80 80"
      width={size}
      height={size}
      aria-hidden="true"
    >
      <circle cx="40" cy="22" r="11" />
      <circle cx="58" cy="40" r="11" />
      <circle cx="40" cy="58" r="11" />
      <circle cx="22" cy="40" r="11" />
    </svg>
  );
}

6 · Uso de tokens no JSX

CSS vars funcionam direto em style JSX. Pra dark mode, troque o data-attr no <html>.

<button
  className="btn btn-primary"
>
  Falar com a gente
</button>

<div style={{
  padding: "var(--sp-8)",
  background: "var(--color-accent-soft)",
  borderRadius: "var(--r-md)",
}}>
  ...
</div>

Notas: (1) Não precisa de Tailwind nem CSS-in-JS — tokens via var() + classes de primitivos cobrem 95% dos casos. (2) Pra otimização, considere next/font/local em vez do @font-face manual — o resultado visual é idêntico. (3) Hash-based asset versioning (querystring) pode ser feito por middleware ou pelo próprio Vercel se quiser cache busting.
